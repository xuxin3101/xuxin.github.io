spring mvc的简单运行流程

spring mvc是怎么运行的呢，我们配置完之后会明白所谓spring mvc也就是把我们程序的控制权，请求解析等等控制权交给DispatcherServlet，所以本质上，他还是基于servlet运行的，但是我们一般会让这个DispatcherServlet拦截我们所有的请求，也就是说，所有的请求全部交给spring去处理，了解到这里之后我们应该明白，改如何配置。

spring mvc配置

因为我们知道了这只是一个servlet，但是要拦截所有的操作，所以我们应该会在web.xml中写正常servlet的配置

  • 里面的参数很简单，也有很多固定的，有一个init-param可以看得出，这是spring mvc的配置文件，这里只讲原理，实践部分会放到后面去讲解。
    当完成了配置，当我们的web应用获取到请求时，会交由DispatcherServlet去处理，这个时候，DispatcherServlet依赖的配置会去springmvc-config.xml中读取，也就是会去这里面读取有没有对应这个请求地址的controller，如果有，交由这个控制器相关的方法处理，处理完了之后会return，传统的mvc会return一个ModelAndView，也就是会吧model和view封装好了一起返回回去，然后view可以去调用model中的数据，完成页面的渲染，最后返回给用户。
  • 上面全部是本人的个人理解，不保证权威，但是希望能给读者一些启迪，尽信书不如无书，笔者愿意共同交流共同进步。
